# 🚀 TrendTracker 개발 로드맵 (Development Roadmap)

본 문서는 TrendTracker 앱의 사용자 경험(UX) 향상과 기능 확장을 위한 단계별 개선 계획을 담고 있습니다.

## 🎨 1. 디자인 및 사용자 인터페이스 (Design & UI/UX)

### 1.1 핀터레스트 스타일 레이아웃 도입

- **목표**: 기사 목록을 단순 리스트 형태에서 카드형 그리드 레이아웃으로 변경 (Pinterest Masonry Look).
- **상세**:
  - `st.columns` 또는 커스텀 CSS를 활용하여 카드 디자인 구현.
  - 각 카드에 이미지(가능한 경우), 제목, 요약, 출처, 날짜 포함.
  - 마우스 호버 시 그림자 효과 등 미이크로 인터랙션 추가.

- **url**: https://kr.pinterest.com

### 1.2 검색 및 컨트롤 레이아웃 최적화

- **검색창 인라인 배치**: '트렌드 분석 키워드' 입력창과 '분석 시작' 버튼을 한 줄(Horizontal)로 배치하여 공간 효율성 증대.
- **날짜 정보 강조**: 기사 헤드라인 바로 아래 또는 카드 하단에 발행 날짜를 눈에 띄게 표시하여 정보의 최신성 파악 용이화.
- **아이콘 및 에셋 교체**: 기존 '밤티' 아이콘을 앱의 성격(트렌드, 뉴스, 분석)에 맞는 세련된 로고/아이콘 세트로 교체.

## ✨ 2. 주요 기능 확장 (Feature Enhancements)

### 2.1 정밀한 뉴스 검색 및 필터링

- **카테고리 지정 검색**: 과학, 의학, 공학, 사회, 문화, 예술, 스포츠, 경제, 정치 등 카테고리 선택 기능 추가. (검색어와 조합하여 검색 정확도 향상)
- **날짜 범위 설정**: 특정 기간(최근 1일, 1주일, 1개월 등) 내의 기사만 보기 또는 날짜 지정 필터링 기능.
- **검색 결과 수 확대**: 기본 검색 건수(5~10건)를 확장하여 더 광범위한 트렌드 분석 가능하도록 옵션 제공.
- **언론사 다양화**: Tavily API 설정을 조정하여 특정 도메인에 치우치지 않고 다양한 신뢰할 수 있는 매체의 기사를 수집.

### 2.2 검색 편의성 개선

- **엔터키(Enter) 입력 지원**: 버튼 클릭 없이 검색어 입력 후 엔터키만으로 바로 분석이 시작되도록 로직 개선.
- **검색어 유연성**: 키워드 내 띄어쓰기가 포함되어도 정확한 검색이 이루어지도록 전처리 로직 강화.

## 🛠️ 3. 기술적 개선 및 성능 최적화 (Technical Tasks)

### 3.1 검색 품질 고도화

- **멀티 가용 소스**: Tavily 외에 추가적인 뉴스 API(예: NewsAPI, Google Search API) 연동 검토.
- **검색어 엔진 최적화**: 사용자가 입력한 키워드를 AI가 분석하여 더 나은 검색 쿼리로 자동 변환하는 기능 (Query Expansion).

### 3.2 성능 및 안정성

- **이미지 로딩 최적화**: 카드 레이아웃 적용 시 기사 썸네일 이미지를 효율적으로 캐싱하고 로딩하는 로직.
- **에러 핸들링**: 특정 언론사 접근 제한이나 API 한도 초과 시 더 친절한 유저 메시지 제공.
